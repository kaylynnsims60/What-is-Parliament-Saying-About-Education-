# Hansard Education Speech Project

Code, documentation, and written materials for a final project analyzing how education is discussed in Canadian House of Commons debate, 1901–2019.

## Project Overview

This project examines how Members of Parliament discuss education in House of Commons debate and what patterns may emerge over time. Using Hansard speech data, hand-coded validation data, and a code-based workflow, the project identifies education-related speeches, classifies major themes, and generates descriptive and analytic outputs. The repository contains the project proposal, supporting documentation, code files, outputs, and final essay.

## Data Note

The full raw Hansard corpus and several large intermediate CSV files were too large to store directly in this GitHub repository. This repository therefore includes the full code pipeline used to transform the data, along with documentation explaining how the raw data were obtained and how large processed files can be regenerated locally.

Instructions for accessing and placing the raw data are provided in `Data/raw/README.md`. Instructions for regenerating large processed or derived files are provided in the relevant subfolders in `Data/`.

## Workflow

The project workflow proceeds in the following order:

1. Download or otherwise obtain the raw Hansard data used for the project and place those files in `Data/raw/` as described in `Data/raw/README.md`.
2. Run the data preparation scripts in `Code/` to import the raw files, retain the relevant metadata, and build the working corpus.
3. Run the filtering scripts to identify speeches that reference education and create the education-related corpus used in the project.
4. Where applicable, run the Google Colab scripts in `Code/Google Colab Scripts/` for zero-shot classification and classification-model development.
5. Run the validation scripts together with the hand-coded sample files in `Data/hand_coded/` to assess the coding or classification approach.
6. Run the main analysis scripts to examine education-related speech across time, parties, and topic categories.
7. Run the figure and table scripts to generate the outputs stored in `Outputs/`.
8. Consult the final paper in `Paper/` for the written interpretation of the results.

## Notes on Reproducibility

Because some source and intermediate data files were too large for GitHub, the repository is designed to document the full workflow and allow large files to be recreated locally from the public source data. File-specific instructions are provided in the relevant `Data/` subfolder README files.
