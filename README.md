# Hansard Education Speech Project

Code, documentation, and written materials for a project analyzing how education is discussed in Canadian House of Commons debate.

## Project overview

This project examines how Members of Parliament discuss education in House of Commons debate and what patterns speech may reveal over time. The project uses Hansard speech data, hand-coded validation data, and a code-based workflow to identify education-related speeches, classify seven themes, and generate descriptive and analytic images. The repository contains the project proposal, codebook, transformation scripts, outputs, and final essay.

## Data note

The full raw Hansard corpus and several large intermediate CSV files were too large to store directly in this GitHub repository. This repository therefore includes:
- documentation for the raw data source and file placement
- the full code pipeline used to transform the raw data
- the project outputs and final written materials

Instructions for raw data access and placement are provided in `Data/raw/README.md`. Instructions for regenerating large processed files are provided in `Data/processed/README.md` and/or `Data/derived/README.md`.

## Workflow

The project workflow proceeds in the following order:

1. Download or otherwise obtain the raw Hansard data used for the project and place those files in `Data/raw/` as described in `Data/raw/README.md`.
2. Run the data preparation scripts in `Code/` to import the raw files, retain the relevant metadata, and build the working corpus.
3. Run the filtering scripts to identify speeches that reference education and create the education-related corpus used in the project.
4. Run the validation scripts together with the hand-coded sample files in `Data/hand_coded/` to assess the coding or classification approach.
5. Run the main analysis scripts to examine education-related speech across time, parties, and topic categories.
6. Run the figure and table scripts to generate the outputs stored in `Outputs/`.
7. Consult the final paper in `Paper/` for the written interpretation of the results.

## Code workflow

The `Code/` folder contains the scripts used to transform the raw data into figures. These scripts are intended to be run in sequence. Where possible, script names are numbered to reflect workflow order.

## Notes on reproducibility

Because some source and intermediate data files were too large for GitHub, the repository is designed to document the full workflow and allow large files to be recreated locally from the public source data. File-specific instructions are provided in the relevant subfolder README files.
